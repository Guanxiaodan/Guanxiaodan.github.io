# Typescript 的枚举类型

一组具有名字的常量集合

# 数字枚举

有反向映射,既可以用 value 索引，也可以用 key 索引,原理是反向映射 (const p = {}; p[p['m']=3]='m')

  • 没有初始值，从上往下，从 0 开始递增
enum Role {
  Reporter,
  Developer,
  Maintainer,
  Owner,
  Guest,
}
console.log(Role.Reporter); // 0

枚举成员只读，定义后不允许修改

# 常量枚举 （const）

  • 常量枚举分三种情况
    • 没有初始值
    • 对已有枚举成员的引用
    • 常量表达式
  • 常量枚举成员会在编译时候计算出结果，然后以常量的形式出现在运行时环境
  • 用 const 声明的枚举就是常量枚举
  • 特点：会在编译阶段被移除，即没有编译后的代码,可以使用下面的这个 enum Month 在官网 (opens new window)试试
  • 使用场景：当不需要使用一个对象，而需要使用对象的值得时候，这样会减少在编译环境的代码，如下：
const enum Month {
  Jen,
  Feb,
  Mar,
}
let month = [Month.Jen, Month.Feb, Month.Mar];

# 需要被计算的枚举成员 (computed)

  • 需要被计算的枚举成员
    • 是一些非常量的表达式
    • 不会在编译阶段进行计算，而是会被保留到程序的执行阶段再进行计算

# 枚举类型

枚举和枚举成员，都可以作为一种单独的类型存在

  • 有三种情况
    • 1.枚举成员没有任何初始值（其实也相当于数字枚举）
    • 2.所有成员都是数字枚举
    • 3.所有成员都是字符串枚举
  • 前面两种情况，可以把数值赋值给变量，取值也可以超出枚举成员的定义
  • 两种不同类型的枚举不能进行比较

枚举成员没有任何初始值（其实也相当于数字枚举）

enum E {
  a,
  b,
}
let e: E = 3;
let e1: E.a;
let e2: E.b;
let e3: E.a;
// 两种不同类型的枚举不能进行比较，如下：
// e1===e2 会报错：“此条件将始终返回 "false"，因为类型 "E.a" 和 "E.b" 没有重叠”
// 因为e1和e2属于两种不同的类型
